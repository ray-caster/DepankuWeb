Color Psychology

  • Blue (Primary): Trust, security, and professionalism - essential for educational platforms
  • Green (Secondary): Growth, learning, and nature - represents progress and development
  • Orange (Accent): Energy, creativity, and innovation - sparks curiosity and engagement
  • Purple (Accent): Wisdom, creativity, and technology - represents innovation and forward-thinking
  • Grays (Neutral): Clarity, professionalism, and readability - provides balance and sophistication

Typography System

Our typography system is designed for maximum readability and modern aesthetics, with fluid scaling for all screen sizes.

Heading 1 - Display

font-size: clamp(2.25rem, 5vw, 3.75rem); font-weight: 800; line-height: 1.1

A modern, tech-forward display font for maximum impact

Heading 2 - Section

font-size: clamp(1.875rem, 4vw, 3rem); font-weight: 700; line-height: 1.2

Perfect for section titles and important headings

Heading 3 - Subsection

font-size: clamp(1.5rem, 3vw, 2.25rem); font-weight: 600; line-height: 1.3

Ideal for subsections and card titles

Heading 4 - Small Heading

font-size: clamp(1.25rem, 2.5vw, 1.875rem); font-weight: 600; line-height: 1.4

Great for smaller headings and labels

Heading 5 - Label

font-size: 1.25rem; font-weight: 600; line-height: 1.4

Perfect for form labels and metadata

Heading 6 - Caption

font-size: 1.125rem; font-weight: 600; line-height: 1.4

Used for captions and small text

Body Text - Regular

font-size: 1rem; font-weight: 400; line-height: 1.6

This is the standard body text with optimal readability for long-form content. The line height and font weight are carefully chosen to reduce eye strain and improve comprehension.

Small Text - Metadata

font-size: 0.875rem; font-weight: 400; line-height: 1.5

Used for metadata, timestamps, and supplementary information.

Accessibility

Our design system is built with accessibility as a core principle, ensuring compliance with WCAG 2.1 AA standards.

Color Contrast

All text meets minimum contrast ratios of 4.5:1 for normal text and 3:1 for large text.

Primary Blue on White
Ratio: 6.4:1
✓ WCAG AA Compliant
Dark Gray on Light Gray
Ratio: 4.8:1
✓ WCAG AA Compliant

Keyboard Navigation

All interactive elements are fully keyboard accessible with proper focus indicators.

  • ✓ Tab order follows logical sequence
  • ✓ Focus states are clearly visible
  • ✓ Keyboard shortcuts are available
  • ✓ Screen reader support is implemented

Screen Reader Support

ARIA labels and roles are used to ensure compatibility with assistive technologies.

  • ✓ Semantic HTML elements are used
  • ✓ ARIA labels provide context
  • ✓ Hidden text for screen readers
  • ✓ Form labels are properly associated

Reduced Motion

Animations respect user preferences for reduced motion.

  • ✓ Prefers-reduced-motion media query
  • ✓ Alternative styles for motion sensitivity
  • ✓ No essential information conveyed through motion
  • ✓ Smooth transitions instead of animations

Usage Guidelines

Best practices for using the design system effectively.

🎨

Consistency

Use the predefined components and styles consistently throughout your application to maintain a cohesive user experience.

♿

Accessibility First

Always consider accessibility when implementing new features. Test with keyboard navigation and screen readers.

📱

Mobile-First Design

Design for mobile devices first, then enhance for larger screens. This ensures a better experience for all users.

⚡

Performance Optimization

Use CSS custom properties for theming and optimize animations for smooth performance across all devices.

🔍

Testing

Test your implementation across different browsers, devices, and assistive technologies to ensure compatibility.
